## Управление сервисами

### `POST /services` — Создать сервис

**Статус:** `201 Created`

**Тело запроса:**
```json
{
  "name": "Платёжный шлюз",
  "description": "Основной сервис обработки платежей"
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `name` | `string` | да | Название сервиса |
| `description` | `string` | нет | Описание сервиса |

**Ответ `201`:**
```json
{
  "id": 1,
  "name": "Платёжный шлюз",
  "description": "Основной сервис обработки платежей",
  "created_at": "2026-04-17T10:00:00Z"
}
```

**Коды ошибок:** `422` — невалидное тело запроса

---

### `GET /services` — Список всех сервисов

**Статус:** `200 OK`

**Параметры запроса:** нет

**Ответ `200`:**
```json
[
  {
    "id": 1,
    "name": "Платёжный шлюз",
    "description": "Основной сервис обработки платежей",
    "created_at": "2026-04-17T10:00:00Z"
  },
  {
    "id": 2,
    "name": "Сервис авторизации",
    "description": null,
    "created_at": "2026-04-17T11:00:00Z"
  }
]
```

---

### `GET /services/{id}` — Получить сервис

**Статус:** `200 OK`

**Path параметры:** `id` — идентификатор сервиса

**Ответ `200`:**
```json
{
  "id": 1,
  "name": "Платёжный шлюз",
  "description": "Основной сервис обработки платежей",
  "created_at": "2026-04-17T10:00:00Z"
}
```

**Коды ошибок:** `404` — сервис не найден

---

### `DELETE /services/{id}` — Удалить сервис

**Статус:** `204 No Content`

**Path параметры:** `id` — идентификатор сервиса

Каскадно удаляет все эндпоинты, историю проверок, ответственных и SLA-конфиг сервиса.

**Коды ошибок:** `404` — сервис не найден

## Управление эндпоинтами

### `POST /services/{id}/endpoints` — Добавить эндпоинт

**Статус:** `201 Created`

**Path параметры:** `id` — идентификатор сервиса

**Тело запроса:**
```json
{
  "url": "https://payment.internal/health",
  "is_active": true
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `url` | `string` | да | Полный URL для проверки |
| `is_active` | `boolean` | нет | Включить в проверки (по умолчанию `true`) |

**Ответ `201`:**
```json
{
  "id": 10,
  "service_id": 1,
  "url": "https://payment.internal/health",
  "is_active": true,
  "created_at": "2026-04-17T10:05:00Z"
}
```

**Коды ошибок:** `404` — сервис не найден, `422` — невалидное тело

---

## SLA-конфигурация

### `PUT /services/{id}/sla-config` — Задать целевой SLA

**Статус:** `200 OK`

**Path параметры:** `id` — идентификатор сервиса

Создаёт запись если не существует, обновляет если существует (upsert).

**Тело запроса:**
```json
{
  "target_percent": 99.9
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `target_percent` | `number` | да | Целевой SLA в процентах, диапазон `0.0–100.0` |

**Ответ `200`:**
```json
{
  "id": 3,
  "service_id": 1,
  "target_percent": 99.9
}
```

**Коды ошибок:** `404` — сервис не найден, `422` — значение вне диапазона

---

### `GET /services/{id}/sla-config` — Получить SLA-конфиг

**Статус:** `200 OK`

**Path параметры:** `id` — идентификатор сервиса

**Ответ `200`:**
```json
{
  "id": 3,
  "service_id": 1,
  "target_percent": 99.9
}
```

**Коды ошибок:** `404` — сервис или конфиг не найден

## Статистика и отчёты

### `GET /services/{id}/sla` — SLA сервиса за 30 дней

**Статус:** `200 OK`

**Path параметры:** `id` — идентификатор сервиса

Возвращает SLA по каждому эндпоинту сервиса за скользящие 30 дней.

**Ответ `200`:**
```json
{
  "service_id": 1,
  "service_name": "Платёжный шлюз",
  "target_percent": 99.9,
  "period_days": 30,
  "endpoints": [
    {
      "endpoint_id": 10,
      "url": "https://payment.internal/health",
      "sla_percent": 99.72,
      "total_checks": 1440,
      "up_checks": 1436,
      "is_meeting_target": false
    },
    {
      "endpoint_id": 11,
      "url": "https://payment.internal/api/status",
      "sla_percent": null,
      "total_checks": 0,
      "up_checks": 0,
      "is_meeting_target": null
    }
  ]
}
```

> `sla_percent: null` означает, что проверок за период не было — не путать с `0`.  
> `is_meeting_target: null` — когда `sla_percent` равен `null`.

**Коды ошибок:** `404` — сервис не найден

---

### `GET /services/sla-summary` — Сводка SLA по всем сервисам

**Статус:** `200 OK`

**Ответ `200`:**

[
  {

    "service_name": "Платёжный шлюз",

    "worst_endpoint_sla": 99.72,

### `GET /services/{id}/history` — История проверок эндпоинтов сервиса

**Статус:** `200 OK`

**Path параметры:** `id` — идентификатор сервиса

**Query параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `endpoint_id` | `integer` | нет | Фильтр по конкретному эндпоинту |
| `limit` | `integer` | `100` | Максимальное количество записей |
| `offset` | `integer` | `0` | Смещение для пагинации |
| `from` | `datetime` | нет | Начало периода (ISO 8601) |
| `to` | `datetime` | нет | Конец периода (ISO 8601) |

**Ответ `200`:**
```json
{
  "service_id": 1,
  "total": 1440,
  "limit": 100,
  "offset": 0,
  "items": [
    {
      "id": 98765,
      "endpoint_id": 10,
      "url": "https://payment.internal/health",
      "checked_at": "2026-04-17T12:00:00Z",
      "is_available": true,
      "status_code": 200,
      "response_time_ms": 45,
      "error_message": null
    },
    {
      "id": 98700,
      "endpoint_id": 10,
      "url": "https://payment.internal/health",
      "checked_at": "2026-04-17T11:59:00Z",
      "is_available": false,
      "status_code": null,
      "response_time_ms": null,
      "error_message": "Connect timeout"
    }
  ]
}
```

**Коды ошибок:** `404` — сервис не найден, `422` — невалидные параметры

### `GET /services/dashboard` — Общий дашборд

**Статус:** `200 OK`

Текущий статус всех сервисов: последняя проверка каждого активного эндпоинта.

**Ответ `200`:**
```json
{
  "generated_at": "2026-04-17T12:00:00Z",
  "services": [
    {
      "service_id": 1,
      "service_name": "Платёжный шлюз",
      "overall_status": "DOWN",
      "endpoints": [
        {
          "endpoint_id": 10,
          "url": "https://payment.internal/health",
          "status": "UP",
          "last_checked_at": "2026-04-17T12:00:00Z",
          "last_status_code": 200,
          "last_response_time_ms": 45
        },
        {
          "endpoint_id": 11,
          "url": "https://payment.internal/api/status",
          "status": "DOWN",
          "last_checked_at": "2026-04-17T11:59:00Z",
          "last_status_code": 500,
          "last_response_time_ms": null
        }
      ]
    }
  ]
}
```

> `overall_status: "DOWN"` — если хотя бы один активный эндпоинт в статусе `DOWN`.  
> `overall_status: "UNKNOWN"` — если ни одной проверки ещё не было.

## Управление мониторингом

### `POST /monitoring/start` — Запустить планировщик

**Статус:** `200 OK`

Запускает `APScheduler`, если он не запущен. При старте приложения планировщик запускается автоматически.

**Ответ `200`:**
```json
{
  "status": "started",
  "check_interval_seconds": 60
}
```

**Коды ошибок:** `409` — планировщик уже запущен

---

### `POST /monitoring/stop` — Остановить планировщик

**Статус:** `200 OK`

Останавливает `APScheduler`. Текущие выполняющиеся проверки завершатся штатно.

**Ответ `200`:**
```json
{
  "status": "stopped"
}
```

**Коды ошибок:** `409` — планировщик уже остановлен

---

### `POST /monitoring/trigger` — Запустить проверку вручную

**Статус:** `200 OK`

Немедленно запускает внеплановый цикл проверок всех активных эндпоинтов.

**Тело запроса:** нет

**Ответ `200`:**
```json
{
  "status": "triggered",
  "endpoints_checked": 5,
  "started_at": "2026-04-17T12:00:00Z"
}
```

---

### `GET /monitoring/status` — Статус планировщика

**Статус:** `200 OK`

**Ответ `200`:**
```json
{
  "scheduler_running": true,
  "check_interval_seconds": 60,
  "next_run_at": "2026-04-17T12:01:00Z",
  "active_endpoints_count": 5
}
```

## Общие соглашения

### Форматы

| Параметр | Формат |
|----------|--------|
| Даты/время | ISO 8601 с таймзоной: `2026-04-17T12:00:00Z` |
| Числа SLA | `NUMERIC(5,2)`: `99.9`, `100.0` |
| Null-значения | Явный `null` в JSON, не отсутствующее поле |

### Коды ответов

| Код | Смысл |
|-----|-------|
| `200` | Успешный запрос |
| `201` | Ресурс создан |
| `204` | Успешно, без тела ответа |
| `404` | Ресурс не найден |
| `409` | Конфликт состояния |
| `422` | Ошибка валидации входных данных |

### Формат ошибки `422`

```json
{
  "detail": [
    {
      "loc": ["body", "target_percent"],
      "msg": "ensure this value is less than or equal to 100",
      "type": "value_error.number.not_le"
    }
  ]
}
```

### Формат ошибки `404`

```json
{
  "detail": "Service not found"
}
```
